# Function Calling

Function Calling 是 LLM 的一种能力——模型输出结构化的函数调用参数，而不是直接执行代码。它让 LLM 能够与外部系统和工具交互，是构建 AI Agent 的核心基础。

## 什么是 Function Calling

传统上，LLM 只能输出文本。Function Calling 扩展了这一能力：开发者为 LLM 定义一组可调用的函数（含名称、描述、参数 Schema），LLM 根据用户需求决定是否调用某个函数，并以 JSON 格式输出调用参数。真正的函数执行在本地或远端完成，LLM 本身不执行代码。

## 工作流程

1. **定义函数** — 编写函数定义，包括函数名、功能描述、参数列表及其 JSON Schema（类型、必填字段、枚举约束等）。
2. **发送给 LLM** — 将函数定义作为请求的一部分发给模型。
3. **LLM 决策** — 模型判断是否需要调用函数。如果需要，输出结构化参数 JSON。
4. **本地执行** — 在应用侧执行该函数（例如查数据库、调用 API、执行计算）。
5. **结果回传** — 将函数执行结果返回给 LLM，模型基于此生成最终回复。

## 结构化输出模式

Structured Outputs（JSON Mode）是 Function Calling 的进阶形式，它强制要求 LLM 的输出严格遵循给定的 JSON Schema，适用于需要高可靠性格式的场景。OpenAI、Anthropic 等主流厂商均支持此模式。

## 应用场景

- **AI 测试用例生成**：指定输出 JSON Schema，要求 LLM 生成符合预设字段结构的测试用例集，可直接导入自动化测试框架。
- **AI Bug 分类**：定义 `category`（枚举）、`severity`（1-5）、`description`（字符串）等固定字段，保证每次输出结构一致，方便后续的自动化处理和统计分析。
- **信息抽取**：从非结构文本中提取结构化数据，填入预定义字段。
- **工具编排**：Agent 根据任务需求动态调用多个函数，组合成完整工作流。

## 优势

Function Calling 将 LLM 的语言理解能力与外部系统的执行能力解耦，既利用了 LLM 的语义理解优势，又避免了 LLM 直接执行代码的安全隐患。
