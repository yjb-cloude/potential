# RAG Pattern

RAG 即 Retrieval Augmented Generation（检索增强生成），是一种通过在生成过程中引入外部知识检索来增强 LLM 能力的技术架构。

## 为什么需要 RAG

LLM 存在三个天然局限：知识截止于训练数据日期、缺乏专业领域知识（如内部文档、私有代码库）、以及可能产生幻觉（Hallucination）。RAG 通过实时检索外部知识库，为 LLM 提供事实依据，显著降低幻觉、提升回答准确性。

## 核心流程

```
文档 → 分块(Chunking) → Embedding → 向量库 → 检索(相似度) → 注入Prompt → LLM生成
```

1. **文档分块（Chunking）** — 将长文档切分为语义完整的片段。
2. **Embedding** — 将文本块转换为向量（dense vector），存储在向量数据库中。
3. **检索** — 用户提问时，将问题同样转为向量，在向量库中搜索语义最相似的文本块。
4. **注入 Prompt** — 将检索到的文本块作为上下文注入到 Prompt 中。
5. **LLM 生成** — LLM 基于检索到的上下文和原始问题生成最终回答。

## 分块策略

- **按段落分块**：保持自然语义边界，适用于结构良好的文档。
- **固定大小分块**：按固定 token 数切分，简单可控。
- **语义分割**：使用 NLP 模型检测语义边界，更精准但计算成本高。
- **Chunk Overlap**：相邻块之间保留重叠内容，避免关键信息被切断。

## Embedding 与向量数据库

- **Embedding 模型**：将文本映射到高维语义向量空间，使语义相似的文本在向量空间中距离相近。常用模型包括 text-embedding-3-small、bge 等。
- **向量数据库**：专门存储和检索向量的系统。常用方案有 Milvus（分布式）、Pinecone（云服务）、Chroma（轻量）、FAISS（本地高性能）。

## 检索方式

- **相似度搜索（Semantic Search）**：纯向量检索，找到语义最接近的 Top-K 结果。
- **混合检索（Hybrid Search）**：结合关键词匹配（BM25）和向量相似度，综合排序。

## 简化实现

小规模场景（个人项目、Demo）无需搭建复杂的向量数据库，用 JSON 文件存储文本块及其向量，在内存中做余弦相似度搜索即可。

## 应用场景

- **AI Bug 分类**：检索历史 Bug 库中的相似案例，辅助 LLM 做出更准确的分类决策。
- **智能知识库问答**：基于内部文档、规范、最佳实践构建问答系统。
- **代码搜索与解释**：检索代码库中相关实现，帮助 LLM 理解项目上下文。
